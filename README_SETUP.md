# Fox Snake — настоящая сетка контрибуций

Лиса ходит по реальной сетке ваших контрибуций (52 недели × 7 дней) и съедает
клетки **от наименьшего уровня активности к наибольшему** — та же логика, что
у оригинальной змейки (Platane/snk), только со скином вашей лисы.

## Структура файлов

```
fox-snake-grid/
├── generate_fox_snake.py          # генератор анимации
├── requirements.txt
├── assets/fox_frames/             # 8 кадров бега (из вашего PixExact GIF)
│   ├── frame_0.png ... frame_7.png
└── .github/workflows/
    └── fox-snake.yml              # запускает генератор раз в день
```

## Установка

1. Распакуйте архив и скопируйте **все файлы и папки** в корень репозитория `e24x5-Fox`
   (тот же, где README).
2. Закоммитьте и запушьте.
3. Во вкладке **Actions** репозитория найдите workflow **"Generate Fox Snake"**
   → нажмите **Run workflow** для первого запуска вручную.
4. После успешного выполнения в корне репозитория появится файл `fox-snake.gif`.
5. Вставьте в README:
```markdown
![Fox eating contributions](./fox-snake.gif)
```

Дальше workflow будет обновлять анимацию сам раз в сутки (по расписанию в `fox-snake.yml`).

## Как это работает

- `fetch_contributions()` забирает ту же публичную страницу с сеткой контрибуций,
  что использует оригинальный snk (`github.com/users/<user>/contributions`),
  без токенов и авторизации.
- У каждой клетки есть уровень активности (0–4, как в official Contribution Graph).
- Клетки сортируются по `(уровень, дата)` по возрастанию — отсюда и эффект
  "от малого к большему".
- Лиса на каждом шаге анимации становится в клетку, крашит её в серый ("съедает"),
  и использует ваши настоящие 8 кадров бега по кругу.

## Настройки, которые можно поменять

В `generate_fox_snake.py`:
- `CELL`, `GAP` — размер клеток сетки
- `FOX_SCALE` — размер лисы относительно клеток
- `STEP_FRAMES` — сколько кадров анимации на одну клетку (больше = плавнее, но тяжелее файл)
- `durations.append(70)` — скорость (мс на кадр)

## Важно про размер файла

Полный проход по ~365 дням даёт GIF около 2 МБ — это нормально для GitHub, но
если хочется легче, уменьшите `FOX_SCALE` или `CELL`, либо прогоните итоговый
файл через [ezgif.com/optimize](https://ezgif.com/optimize).
